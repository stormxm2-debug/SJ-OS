# SJ OS — 스태프 폰 로그인 E2E 테스트 가이드

관리자 폰 등록 → 최초 비밀번호 설정 → 번호+비밀번호 로그인 → 역할 메뉴 → 공지사항까지
end-to-end 확인 절차입니다. (자세한 배포는 `SUPABASE_EDGE_FUNCTION_DEPLOYMENT_GUIDE.md`,
공지는 `SUPABASE_ANNOUNCEMENTS_GUIDE.md`.)

## 사전 준비 (Supabase 설정 시)

1. SQL 적용: 기본 스키마 → 관리자 폰 로그인 스키마/RLS → 공지 스키마/RLS.
2. Edge Function 배포: `claim-phone-account`, `request-phone-password-reset`
   (service_role은 함수 secret에만).
3. 프론트엔드 env: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` (anon만).

## 로그인 E2E

1. 대표/관리자 로그인 → **직원 로그인 관리**에서 직원 번호 등록.
2. 직원이 로그인 화면에서 번호 입력 → **최초 비밀번호 설정** → 서버 함수가 계정/프로필 생성.
3. 번호 + 비밀번호로 로그인 → 프로필 역할에 맞는 메뉴 표시.
4. 미등록 번호 → "등록된 직원만 이용할 수 있습니다." / 비활성 → "비활성 직원 계정입니다."
5. 비밀번호 찾기 → 일반 메시지("등록된 직원이면 관리자에게 …") + 관리자 승인.

## 로그인 후 공지사항 확인

6. 관리자: **공지사항 관리**에서 전체/역할별/팀별 공지 작성 후 **게시**.
7. FC 로그인 → **공지사항** 진입 → 본인 대상 게시 공지만 보이는지 확인.
   - 전체 공지: 모두에게 보임.
   - 역할 공지: 역할 일치 직원만.
   - 팀 공지: 팀 일치 직원만.
   - draft/hidden/archived: 직원에게 안 보임.

## 공지 읽음 처리 확인

8. FC가 공지를 열면 읽음 처리 → 안읽음 수 감소 (본인 읽음 기록만 생성).
9. 홈 화면 **최근 공지** 위젯의 안읽음 배지가 갱신되는지 확인.

## FC 공지관리 접근 차단

10. FC 계정으로 **공지사항 관리**(admin 라우트) 접근 시 "접근 권한이 없습니다." 확인.
11. 모바일에서 개발/릴리즈/배포/직원·공지 관리 메뉴가 보이지 않는지 확인.

## 보안 점검

- 렌더러에 service_role 없음, `.env` 미변경.
- 로그에 phone/password/token/session/고객 데이터 없음.
- RLS 활성 유지 (공개 조회 정책 없음, 읽음 기록은 본인만).

## 로컬 데모(미설정) 모드

- Supabase env 없이도 로컬 데모 로그인 + 로컬 공지로 화면 흐름을 확인할 수 있습니다.
- 실제 데이터/권한은 Supabase 연결 후 RLS로 최종 적용됩니다.
