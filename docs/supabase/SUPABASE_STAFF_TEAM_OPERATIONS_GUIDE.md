# SJ OS — 직원 / 팀 운영 관리 가이드

대표/관리자가 조직 구조(팀, 팀장, 역할, 배정, 상태)를 관리하는 방법입니다. **FC/팀장은
접근할 수 없습니다.** 실제 접근 제어는 RLS가 최종 적용합니다.

## 화면 구성

- **직원 / 팀 관리** (`staff-team`): 조직 구조 관리 (팀/역할/배정/상태).
- **직원 로그인 관리** (`staff-login`): 허용 휴대폰 번호 등록 (로그인 접근 제어).
- 둘 다 대표/관리자 전용. (FC/팀장 → "접근 권한이 없습니다.")

## 절차

1. **팀 생성** — 직원 / 팀 관리 → 새 팀명 입력 → 팀 생성 (`public.teams` insert).
2. **직원 번호 등록** — 직원 로그인 관리에서 휴대폰 번호를 먼저 등록
   (`staff_login_accounts`).
3. **직원 팀 배정** — 직원 관리에서 팀 드롭다운 선택
   (`staff_login_accounts.team_id`, 프로필 연결 시 `profiles.team_id`도 갱신).
4. **팀장 지정** — 팀장 지정 카드에서 직원 선택 → 확인. "팀장 지정 시 해당 직원의 역할이
   팀장으로 변경될 수 있습니다." (FC였다면 team-leader로 승격; owner/admin은 변경하지 않음).
5. **역할 변경** — 직원 행의 역할 드롭다운. **대표만 owner를 지정**할 수 있습니다(관리자는
   owner 옵션이 보이지 않음).
6. **비활성화 / 차단** — 직원 행 버튼. `staff_login_accounts.status` 갱신, 프로필 연결 시
   `profiles.status`도 매핑(active→active, 그 외→inactive).
7. **비밀번호 상태 확인** — 미설정 / 설정됨 / 재설정 요청 / 승인.
8. **프로필 연결 확인** — 프로필 연결됨 / 미연결 (연결은 최초 비밀번호 설정 서버 함수가 수행).

## 관계

- `staff_login_accounts` = 로그인 허용 게이트(휴대폰).
- `profiles` = 로그인한 사용자(역할/팀/상태, RLS의 기준).
- `teams` = 팀(팀장 leader_id).
- 최초 비밀번호 설정 시 서버 함수가 Auth 사용자 + profiles 행을 생성/연결합니다.

## 권한별 접근

- 대표/관리자: 전체 관리.
- 팀장: 이번 스프린트에서 직원/팀 관리 불가 (라우트 접근 시 접근 거부).
- FC: 접근 불가.

## RLS 주의사항

- teams/profiles/staff_login_accounts 관리는 owner/admin만. FC는 전체 프로필/번호 목록을
  볼 수 없습니다. 공개(public) 조회 정책 금지. RLS 비활성화 금지.
- 렌더러는 service_role을 사용하지 않으며, Auth 사용자 생성/비밀번호 설정은 서버 함수에서만
  수행합니다.
